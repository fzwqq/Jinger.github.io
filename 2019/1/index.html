  • 阅读/调研

    • Topic 1： $\mu{RTS}$
      • 相关研究进展：
      • 可跟进项：
    • Paper 1: Attention is all you need. Vaswani, A., Shazeer, N., Parmar, N., Uszkoreit, J., Jones, L., Gomez, A. N., … & Polosukhin, I. (2017).
      • 内容：the Illustrated Transformer , Google AI blog
        • Attention is a concept that helped improve the performance of neural machine translation applications.
      • Why: The Transformer requires less computation to train and is a much better fit for modern machine learning hardware, speeding up training by up to an order of magnitude.
      • How：Without the RNN, The Transformer lends itself to parallelization
    • Paper 2： Portfolio greedy search and simulation for large-scale combat in starcraft. David Churchill and Michael Buro. (2013)
      • 内容： To use hill climbing and accurate playout-based evaluations to efficiently search even the largest combat scenarios.
        • Why: With RTS games vast state and action spaces and real-time constraints, existing AI solutions have been shown to be too slow.
        • How: The algo assumes the existence of a pool of scripts $\Sigma$. A script $\hat{\sigma}$ is a function mapping a game state s and a unit u to a legal action m for u at s. In every game state s in which a unit u is ready to perform an action, a search algorithm selects a script $\hat{\sigma}$ from $\Sigma$ for u. The two scripts were used in PGS which named No-OverKill-Attack-Value (NOK-AV) and Kiter. The algorithm can be broken down into three main parts. Furthermore, PGS uses a playout-based evaluation.
          • In the first part, it set the player’s initial seed policy to the best performing script found via the process that iterate over all scripts, and then perform a playout with each iteration.
          • In the second part, it uses a hill-climbing procedure to search for each script in the scripts set exactly once for each unit. At each iteration, it performs a playout with the individual unit-script assignments and records the result. After performing desired improvement iterations, all units’ script is set to the best one found via the playout process.
          • The third part is to repeatedly improve the two players in turns based on the current stronger opponent via the second procedure.
        • What: PGS uses a playout-based evaluation which would cause incorrectly evaluates action containing non-offensive scripts.It was improved by another hill-climbing search algorithm Stratified Strategy Selection (SSS) designed by Lelis
    • Paper 3： Evolving action abstractions for real-time planning in extensive-form games.Julian RH Marino, Rubens O Moraes, Claudio Toledo, and Levi HS Lelis. （ 2019）
      • 内容：Presented an evolutionary procedure to solve subset selection game (SSG).
      • Why：Instead of using only strategies designed by experts, the algo use a large set of strategies generated from an initial set of expert-designed strategies.
      • How：The Evolutionary algorithm receives as input a set $Z$ of pure strategies from which one can induce action abstractions and the parameters that evolutionary algorithm needs. It returns a subset of $Z$, which is called an individual. The $Z$ set is generated by exchanging the parameters of the expert-designed rule-based strategies.
        • Specifically, they vary the parameters of Rush, Defense, and Military defined by the strategies of Silva, to generate 100 strategies of each type, which results in a $Z$ set of 300 strategies.
        • Furthermore, they use Portfolio Greedy Search (PGS) and Stratified Strategy Selection (SSS) as search algorithms as well as perform two random and independent play-outs with 200 game cycles of length from the state being evaluated according to the built-in SimpleSqrtEvaluationFunction3 function as evaluation function.
      • What：The set $Z$ still need to contained a larger diversity of strategies.
